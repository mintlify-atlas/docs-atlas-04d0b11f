## Overview

Sending events to PingPilot is simple: make a POST request to the events endpoint with your API key and event data. PingPilot will process the event and send notifications to all your configured channels (Discord, Telegram, Email).

## API Endpoint

```
POST https://pingpilot.com/api/v1/event
```

## Authentication

All requests must include your API key in the `Authorization` header as a Bearer token:

```
Authorization: Bearer YOUR_API_KEY
```

<Note>
  Don't have an API key yet? Find it in your [dashboard settings](/guides/api-keys).
</Note>

## Request Format

### Headers

```
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
```

### Body Parameters

<ParamField body="category" type="string" required>
  The name of the event category. Must match an existing category in your account.
  
  **Validation**: Must be lowercase and contain only letters, numbers, or hyphens.
</ParamField>

<ParamField body="description" type="string" optional>
  A description of the event. If not provided, a default description will be generated.
  
  **Default**: `"A new {category} event has occurred!"`
</ParamField>

<ParamField body="fields" type="object" optional>
  Additional data to include with the event. Keys must be strings, values can be strings, numbers, or booleans.
  
  **Example**: `{ "user_id": "12345", "amount": 99.99, "verified": true }`
</ParamField>

<CodeGroup>
```bash cURL

  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "category": "bug",
    "description": "Critical error in payment processor"
  }'
```

```go Go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
    "os"
)

func main() {
    apiKey := os.Getenv("PINGPILOT_API_KEY")
    
    data := map[string]interface{}{
        "category":    "bug",
        "description": "Critical error in payment processor",
    }
    
    jsonData, _ := json.Marshal(data)
    
    req, _ := http.NewRequest("POST", "https://pingpilot.com/api/v1/event", bytes.NewBuffer(jsonData))
    req.Header.Set("Authorization", "Bearer "+apiKey)
    req.Header.Set("Content-Type", "application/json")
    
    client := &http.Client{}
    resp, _ := client.Do(req)
    defer resp.Body.Close()
    
    var result map[string]interface{}
    json.NewDecoder(resp.Body).Decode(&result)
    fmt.Println(result)
}
```

## Advanced Example with Fields

<CodeGroup>
```bash cURL
curl -X POST https://pingpilot.com/api/v1/event \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "category": "sale",
    "description": "New premium plan purchase",
    "fields": {
      "customer_id": "cus_123456",
      "plan": "Premium",
      "amount": 99.99,
      "recurring": true,
      "trial": false
    }
  }'
```

```javascript JavaScript/Node.js
const sendEvent = async (category, description, fields) => {
  try {
    const response = await fetch('https://pingpilot.com/api/v1/event', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.PINGPILOT_API_KEY}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        category,
        description,
        fields
      })
    })

    if (!response.ok) {
      const error = await response.json()
      throw new Error(error.message)
    }

    const data = await response.json()
    console.log('Event sent:', data.eventId)
    return data
  } catch (error) {
    console.error('Failed to send event:', error.message)
    throw error
  }
}

// Usage
await sendEvent('sale', 'New premium plan purchase', {
  customer_id: 'cus_123456',
  plan: 'Premium',
  amount: 99.99,
  recurring: true,
  trial: false
})
```

```python Python
import requests
import os
from typing import Dict, Any, Optional

def send_event(
    category: str,
    description: Optional[str] = None,
    fields: Optional[Dict[str, Any]] = None
) -> Dict[str, Any]:
    """Send an event to PingPilot."""
    api_key = os.environ['PINGPILOT_API_KEY']
    
    payload = {'category': category}
    if description:
        payload['description'] = description
    if fields:
        payload['fields'] = fields
    
    response = requests.post(
        'https://pingpilot.com/api/v1/event',
        headers={
            'Authorization': f'Bearer {api_key}',
            'Content-Type': 'application/json'
        },
        json=payload
    )
    
    response.raise_for_status()
    return response.json()

# Usage
try:
    result = send_event(
        category='sale',
        description='New premium plan purchase',
        fields={
            'customer_id': 'cus_123456',
            'plan': 'Premium',
            'amount': 99.99,
            'recurring': True,
            'trial': False
        }
    )
    print(f"Event sent: {result['eventId']}")
except requests.exceptions.HTTPError as e:
    print(f"Failed to send event: {e.response.json()['message']}")
```
</CodeGroup>

## Response Format

### Success Response

```json
{
  "message": "Event processed successfully",
  "eventId": "clxxx..."
}
```

**Status Code**: `200 OK`

The `eventId` can be used to track the event in your dashboard.

### Error Responses

<AccordionGroup>
  <Accordion title="401 Unauthorized - Missing Authorization">
    ```json
    {
      "message": "Unauthorized"
    }
    ```
    
    **Cause**: No `Authorization` header provided.
    
    **Solution**: Add the `Authorization` header with your API key.
  </Accordion>

  <Accordion title="401 Unauthorized - Invalid Format">
    ```json
    {
      "message": "Invalid auth header format. Expected: 'Bearer [API_KEY]'"
    }
    ```
    
    **Cause**: Authorization header is not in the correct format.
    
    **Solution**: Ensure header is formatted as `Bearer YOUR_API_KEY`.
  </Accordion>

  <Accordion title="401 Unauthorized - Invalid API Key">
    ```json
    {
      "message": "Invalid API key"
    }
    ```
    
    **Cause**: The provided API key doesn't exist or is invalid.
    
    **Solution**: Verify your API key in the dashboard.
  </Accordion>

  <Accordion title="400 Bad Request - Invalid JSON">
    ```json
    {
      "message": "Invalid JSON request body"
    }
    ```
    
    **Cause**: Request body is not valid JSON.
    
    **Solution**: Ensure your request body is properly formatted JSON.
  </Accordion>

  <Accordion title="403 Forbidden - Discord ID Not Set">
    ```json
    {
      "message": "Please enter your discord ID in your account settings"
    }
    ```
    
    **Cause**: Your Discord ID is not configured.
    
    **Solution**: Add your Discord ID in [account settings](/guides/managing-notifications).
  </Accordion>

  <Accordion title="403 Forbidden - Telegram Username Not Set">
    ```json
    {
      "message": "Telegram username not set. Please set your telegram username in your account settings to receive notifications"
    }
    ```
    
    **Cause**: Your Telegram username is not configured.
    
    **Solution**: Add your Telegram username in [account settings](/guides/managing-notifications).
  </Accordion>

  <Accordion title="404 Not Found - Category Doesn't Exist">
    ```json
    {
      "message": "You dont have a category named 'your-category'"
    }
    ```
    
    **Cause**: The specified category doesn't exist in your account.
    
    **Solution**: Create the category first or check for typos.
  </Accordion>

  <Accordion title="422 Unprocessable Entity - Validation Error">
    ```json
    {
      "message": "Validation error details..."
    }
    ```
    
    **Cause**: Request data doesn't meet validation requirements.
    
    **Solution**: Check that:
    - `category` contains only letters, numbers, or hyphens
    - `fields` values are strings, numbers, or booleans
    - All required fields are present
  </Accordion>

  <Accordion title="429 Too Many Requests - Quota Exceeded">
    ```json
    {
      "message": "Monthly quota reached. Please upgrade your plan for more events"
    }
    ```
    
    **Cause**: You've reached your monthly event quota.
    
    **Solution**: Upgrade your plan or wait until next month.
  </Accordion>

  <Accordion title="500 Internal Server Error - Processing Failed">
    ```json
    {
      "message": "Error processing event",
      "eventId": "clxxx..."
    }
    ```
    
    **Cause**: Event was created but notification delivery failed.
    
    **Solution**: Check your notification settings and try again.
  </Accordion>
</AccordionGroup>

## What Happens When You Send an Event?

When PingPilot receives an event:

<Steps>
  <Step title="Authentication">
    Your API key is validated against the database
  </Step>
  
  <Step title="Validation">
    - Request body is validated
    - Category existence is checked
    - Quota limits are verified
  </Step>
  
  <Step title="Event Creation">
    Event is stored in the database with status `PENDING`
  </Step>
  
  <Step title="Notification Delivery">
    Notifications are sent to:
    - **Discord**: As an embedded message
    - **Telegram**: As a formatted text message
    - **Email**: As an HTML email
  </Step>
  
  <Step title="Status Update">
    Event status is updated to `DELIVERED` or `FAILED`
  </Step>
  
  <Step title="Quota Increment">
    Your monthly quota counter is incremented
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Use Environment Variables">
    Never hardcode your API key. Use environment variables:
    
    ```bash
    # .env
    PINGPILOT_API_KEY=your_api_key_here
    ```
  </Accordion>

  <Accordion title="Handle Errors Gracefully">
    Always handle API errors to prevent your application from crashing:
    
    ```javascript
    try {
      await sendEvent('bug', 'Error occurred')
    } catch (error) {
      console.error('Failed to send event:', error)
      // Continue with your application logic
    }
    ```
  </Accordion>

  <Accordion title="Use Descriptive Event Data">
    Provide meaningful descriptions and fields:
    
    ✅ Good:
    ```json
    {
      "category": "payment-failed",
      "description": "Payment declined for subscription renewal",
      "fields": {
        "customer_id": "cus_123",
        "amount": 99.99,
        "reason": "insufficient_funds"
      }
    }
    ```
    
    ❌ Bad:
    ```json
    {
      "category": "error",
      "description": "Something went wrong"
    }
    ```
  </Accordion>

  <Accordion title="Don't Send Sensitive Data">
    Avoid sending passwords, credit card numbers, or other sensitive information in event data.
  </Accordion>

  <Accordion title="Implement Retry Logic">
    For critical events, implement retry logic with exponential backoff:
    
    ```javascript
    async function sendEventWithRetry(data, maxRetries = 3) {
      for (let i = 0; i < maxRetries; i++) {
        try {
          return await sendEvent(data)
        } catch (error) {
          if (i === maxRetries - 1) throw error
          await new Promise(r => setTimeout(r, Math.pow(2, i) * 1000))
        }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Rate Limits & Quotas

PingPilot enforces monthly quotas based on your plan:

- **Free**: 100 events/month
- **Pro**: 1,000 events/month  
- **Enterprise**: Custom limits

Quotas reset on the first day of each month. There are no per-second or per-minute rate limits.
